zookeeper3.4.5安装配置

Zookeeper 分布式服务框架是 Apache Hadoop 的一个子项目，它主要是用来解决分布式应用中经常遇到的一些数据管理问题，如：统一命名服务、状态同步服务、集群管理、分布式应用配置项的管理等。

首先要在安装zookeeper的服务器上安装JAVA运行环境，JAVA运行环境的搭建不多说了。
然后到zookeeper项目首页下载安装包。
假设安装的服务器IP为192.168.1.1，那么将下载的tar包解压到/opt/zookeeper/目录下
解压后修改/opt/zookeeper/zookeeper3.4.5/conf/zoo.cfg配置文件。

单机模式

如果是单机模式安装，配置如下：
tickTime=2000
dataDir=/opt/zookeeper/zkData
clientPort=2181
tickTime：这个时间是作为 Zookeeper 服务器之间或客户端与服务器之间维持心跳的时间间隔，也就是每个 tickTime 时间就会发送一个心跳。

dataDir：顾名思义就是 Zookeeper 保存数据的目录，默认情况下，Zookeeper 将写数据的日志文件也保存在这个目录里。

clientPort：这个端口就是客户端连接 Zookeeper 服务器的端口，Zookeeper 会监听这个端口，接受客户端的访问请求。

通过bin/zkServer.sh start命令启动zookeeper服务。

通过bin/zkServer.sh status查看zookeeper是否启动。
当显示为：
JMX enabled by default
Using config: /opt/zookeeper/zookeeper-3.4.5/bin/../conf/zoo.cfg
Mode: standalone
表示启动成功了。

集群模式

集群模式配置很简单。假设为192.168.1.1，192.168.1.2，192.168.1.3三台机子为例。
首先在单机模式配置的基础上添加以下几个配置
initLimit=5
syncLimit=2
server.1=192.168.1.1:2888:3888
server.2=192.168.1.2:2888:3888
server.3=192.168.1.3:2888:3888

initLimit：这个配置项是用来配置 Zookeeper 接受客户端（这里所说的客户端不是用户连接 Zookeeper 服务器的客户端，而是 Zookeeper 服务器集群中连接到 Leader 的 Follower 服务器）初始化连接时最长能忍受多少个心跳时间间隔数。当已经超过 5 个心跳的时间（也就是 tickTime）长度后 Zookeeper 服务器还没有收到客户端的返回信息，那么表明这个客户端连接失败。总的时间长度就是 5*2000=10 秒

syncLimit：这个配置项标识 Leader 与 Follower 之间发送消息，请求和应答时间长度，最长不能超过多少个 tickTime 的时间长度，总的时间长度就是 2*2000=4 秒

server.A=B：C：D：其中 A 是一个数字，表示这个是第几号服务器；B 是这个服务器的 ip 地址；C 表示的是这个服务器与集群中的 Leader 服务器交换信息的端口；D 表示的是万一集群中的 Leader 服务器挂了，需要一个端口来重新进行选举，选出一个新的 Leader，而这个端口就是用来执行选举时服务器相互通信的端口。如果是伪集群的配置方式，由于 B 都是一样，所以不同的 Zookeeper 实例通信端口号不能一样，所以要给它们分配不同的端口号。

除了修改 zoo.cfg 配置文件，集群模式下还要配置一个文件 myid，这个文件在 dataDir 目录下，这个文件里面就有一个数据就是 A 的值，Zookeeper 启动时会读取这个文件，拿到里面的数据与 zoo.cfg 里面的配置信息比较从而判断到底是那个 server。例如本例子就需要在/opt/zookeeper/zkData目录新建myid文件，里面写入1。

其次将192.168.1.1/opt/zookeeper整个目录通过scp命令COPY到192.168.1.2，192.168.1.3服务器相同路径中：
scp -r /opt/zookeeper root@192.168.1.2:/opt
scp -r /opt/zookeeper root@192.168.1.3:/opt

然后修改/opt/zookeeper/zkData/myid 文件内的值修改为zoo.cfg内配置的值。本例中 192.168.1.2：/opt/zookeeper/zkData/myid 的值为2
192.168.1.3：/opt/zookeeper/zkData/myid 的值为3

最后在各服务器上通过bin/zkServer.sh start命令启动zookeeper服务。
通过bin/zkServer.sh status查看zookeeper是否启动及本节点是leader还是follower
leader:
JMX enabled by default
Using config: /home/solr/zookeeper-3.4.5/bin/../conf/zoo.cfg
Mode: leader
follower:
JMX enabled by default
Using config: /home/solr/zookeeper-3.4.5/bin/../conf/zoo.cfg
Mode: follower

通过以上步骤安装配置成功后就可以使用zookeeper了。具体如果使用请参见官网和其它博客介绍
