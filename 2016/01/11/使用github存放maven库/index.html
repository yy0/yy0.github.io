使用github存放maven库

首先你要先有一个github账号。然后创建一个仓库（例如叫：maven-repo）

1、在本地创建目录：D:\github-repos， 并切换到该目录
2、将远程仓库（maven-repo）colne到本地 git clone https://github.com/{你的github账号名}/maven-repo.git
3、切换到本地导入的仓库目录 pwd: D:\github-repos\maven-repo
4、忽略所有 echo "*" .gitignore
5、提交变更 git commit -m 'ignoring all'
6、为了可以创建分支，首先要提交一些东西在远程仓库中去。因此创建一个空的文件1.txt,然后加入版本控制 加入版本控制 git add 1.txt
7、git commit -m 'test'
9、创建分支 git branch snapshot（用于存放开发中生成的JAR包）， git branch release（存放发布版本的JAR包）
10、git push origin snapshot， git push origin release
11、切换到snapshot分支 git checkout snapshot

本地项目pom.xml配置：
<properties>
  <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
  <local.maven.repository.path>file:D:\github-repos\maven-repo</local.maven.repository.path>
</properties>
<distributionManagement>
  <repository>
    <id>my-maven-repository</id>
    <name>Internal Repository</name>
    <url>${local.maven.repository.path}</url>
  </repository>
</distributionManagement>

当要发布JAR包时，运行maven deploy命令，将项目代码打包发布到本地的仓库（D:\github-repos\maven-repo）里

然后将本地仓库添加到版本控件 git add com\test\0.0.1\
再提交到远程仓库 git commit -m 'test'
最后选择发布到仓库的分支发布jar包 git push origin snapshot

其它项目引用github 仓库里的jar：
项目pom.xml配置：
<repositories>
  <repository>
    <id>my-github-maven-snapshot-repository</id>
    <name>my-github-maven-snapshot-repository</name>
    <url>https://raw.github.com/你的github账号名}/maven-repo/snapshot/</url>
  </repository>
  <repository>
    <id>my-github-maven-release-repository</id>
    <name>my-github-maven-release-repository</name>
    <url>https://raw.github.com/你的github账号名}/maven-repo/release/</url>
  </repository>
</repositories>
