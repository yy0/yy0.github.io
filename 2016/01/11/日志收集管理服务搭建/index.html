ElasticSearch+LogStash+Kibana+Redis搭建日志收集管理服务

一般情况下，一个站点会由多台server构成, 每台server每天会产生多种不同的日志，为了方面分析线上日志，常用做法是把线上日志集中存储到一个地方，然后按照日期，日志类型，日志内关键字等条件查询。而logstash, elasticsearch, 和kibana就是用来做这件事的开源解决方案。

Logstash 运行于Jruby上的日志收集和分析系统，支持多种后端存储，包括elasticsearch, s3, riak, mongodb, redis等

Elasticsearch Java写的搜索引擎，简单好用，很适合日志的存储和查询, 简称ES

Kibana Elasticsearch的查询前端，最早用PHP写的，后来用Ruby重新实现了一遍，最新版本是纯Javascript的，直接在浏览器端运行。

架构上，在服务器不多的情况下，可以使用Logstash分析日志后，直接存储到ES里面，然后调用Kibana做日志查询。

以上对各组件描述直接COPY他人的博客。下面直接进入如何搭建。

logstash,Elasticsearch,Kibana都可以通过这里下载。各安装的服务上都需要安装JRE，其中安装elasticsearch（最新版本1.4.1）的服务器需要JDK8。JAVA运行环境搭建这里就不多说了。

一、elasticsearch1.4.1

1、下载安装包
2、解压到 192.168.1.100:/opt/elasticsearch1.4.1
3、编辑安装目录下config目录内的elasticsearch.yml文件，添加两项目配置：
http.cors.enabled: true
http.cors.allow-origin: http://192.168.1.101:8080
其它配置项目使用默认值。如果不添加这两项目值 Kibana启动访问里会报Connection Failed异常。
http.cors.allow-origin参数的值为kibana服务地址
4、执行：bin/elasticsearch -d 命令启动elasticsearch服务。

二、Kibana3.1.2

1、下载安装包
2、解压到 192.168.1.101:/opt/kibana
3、安装tomcat
4、修改tomcat的server.xml配置文件host节点内设置：
<Context path="/" docBase="/opt/kibana" reloadable="true" />
5、修改/opt/kibana/config.js,设置elasticsearch指向ES的服务地址：
elasticsearch: "http://192.168.1.100:9200",
6、启动tomcat。访问http://192.168.1.101:8080/就能看到kibana页面了。

三、redis

偷懒直接上管网看吧，安装很简单。
假设安装在192.168.1.102上，端口为默认值。

四、logstash1.4.2

1）日志收集server端安装
1、下载安装包
2、解压到 192.168.1.101:/opt/logstash
3、进行解压的目录，创建conf目录，并在conf目录下新建logstash的配置文件server.conf:
input {
  redis {
    host => "192.168.1.102"
    data_type => "list"
    key => "logstash"
    db => 7
  }
}
filter {
  if [type] == "error" { # 过滤指定type值的日志
    multiline {
      pattern => "^\s" # 将多行日志合并成一行（java）
      what => "previous"
    }
  }
}
output {
  elasticsearch {
    bind_host => "192.168.1.100"
    bind_port => "9200"
    host => "192.168.1.100"
    port => "9200"
    protocol => "http"
  }
}
4、以后台方式运行服务 nohup bin/logstash -f conf/server.conf &
2）日志采集client端安装
假设收集192.168.1.110:/opt/logs/test.log文件日志
安装过程与收集端一样。但配置文件要配置成如下：
input {
  file {
    type => "solr-info"
    path => "/opt/solrcloud/solr1/logs/info/info.log"
  }
}
filter {
  grok {
    match => { "message" => "%{COMBINEDAPACHELOG}" }
  }
  date {
    match => [ "timestamp" , "yyyy-MM-dd HH:mm:ss Z" ]
  }
  if [type] == "error" {
    multiline {
      pattern => "^\s"
      what => "previous"
    }
  }
}
output {
  redis {
    host => "10.1.0.208" # server端与client端设值一样
    data_type => "list" # server端与client端设值一样
    key => "logstash" # server端与client端设值一样
    db => 7 # server端与client端设值一样
  }
}

logstash更多配置说明请参见这里
4、以后台方式运行服务 nohup bin/logstash -f conf/server.conf &

往日志文件里输入日志通过192.168.1.101:8080/index.html#/dashboard/file/logstash.json
就能查询到日志了。
